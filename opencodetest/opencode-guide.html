🤖 OpenCode CLI 使用邏輯

Terminal AI Agent · Non-Interactive & Persistent Session 完全攻略

1. 核心觀念：兩種運行模式

互動模式（TUI）

你的視覺化終端，安裝後預設進入，包含彩色介面與提示。須要設定 AI Provider。

opencode                        # 進入互動模式（預設）

非互動模式（Script / One-shot）

用在 CLI script、Cron job、串接流程。搭配 --prompt 一次完成任務。

opencode run --prompt "任務描述" --model opencode/big-pickle

2. Session 持久化 — 核心競爭力

OpenCode 最重要的能力：自帶 SQLite 資料庫儲存對話歷史，每次操作都有 Session ID，可以隨時召回、繼續、分支。

指令說明
opencode session list列出所有 sessions（顯示 ID、標題、最後更新時間）
opencode run --continue繼續上一個 session（不開新對話）
opencode run --session 繼續指定的 session ID
opencode run --forkfork 一份 current session 再繼續（不影響原 session）
opencode export [sessionID]匯出 session 成 JSON（含完整對話歷史）
opencode import 從 JSON 檔案匯入 session

輸出範例：

┌──────────────────────────────────────┬──────────────────────────────┬──────────────────┐
│ ID                                   │ Title                        │ Updated          │
├──────────────────────────────────────┼──────────────────────────────┼──────────────────┤
│ ses_18830d080ffem4aVo5n5xupl2q       │ 建立 HTML 報告               │ 2026-05-30 07:45 │
│ ses_18830d07f5035qiHJT6mlJ6Zbf       │ 修正雷達圖座標               │ 2026-05-30 07:50 │
└──────────────────────────────────────┴──────────────────────────────┴──────────────────┘

3. 實戰流程

單次任務（不需記憶）

opencode run --prompt "在目前目錄建立 index.html" --model opencode/big-pickle

持續對話（三步驟）

1
發送第一個任務，儲存回傳的 Session ID 
opencode run "建立一個 HTML 報告" --model opencode/big-pickle --format json 2>&1 | grep sessionID
2
用 --session 繼續同一個 session 加入新任務 
opencode run "修正雷達圖座標" --session ses_18830d080ffem4aVo5n5xupl2q
3
隨時用 session list 查看狀態 
opencode session list

4. 與 Hermes Agent 整合策略

Hermes 作為協調者，OpenCode 作為專案執行者的分工模式：

# Hermes 下令，OpenCode 執行，結果寫入檔案後回傳 summary
opencode run --prompt "在 opencodetest/ 建立一份技術文件" \
  --dir ~/Desktop/special_topic/opencodetest \
  --model opencode/big-pickle
注意：CLI 模式下 OpenCode 同樣沒有你的 project context，需要在 prompt 裡明確定義目標、目錄、相對路徑。

5. 可用模型（已內建，無需 API Key）

使用 opencode models 查看完整清單，以下為目前標配：

opencode/big-pickle opencode/deepseek-v4-flash-free opencode/mimo-v2.5-free opencode/nemotron-3-super-free

使用範例：--model opencode/big-pickle

輸出範例：

6. 重要 Flags 一覽

Flag說明
--prompt "..."直接給任務（單次模式必備）
--model指定模型，格式：provider/model
--continue繼續上個 session
--session 繼續特定 session
--forkfork session 再 continue（安全分支）
--format jsonJSON 输出（可用於 pipeline 解析）
--dir 指定工作目錄
--title "..."給 session 命名
--file 附加檔案到 prompt
--interactive互動分割模式（終端內多工）

7. 長期部署：Server / Web 模式

如果需要長駐、透過程式呼叫、或分享給團隊：

opencode serve               # 啟動 headless server（背景執行）
opencode web                # 啟動 + 開啟瀏覽器 Web UI
opencode attach       # 附加到執行中的 server

Server 模式適合：需要 Hermes 從腳本定時呼叫、或需要多人共享同一個 OpenCode 實例。

8. 限制與注意

  • 沒有內建專案 context — prompt 必須自己包含目錄結構、要創建的檔案、相對路徑
  • CLI 模式仍是獨立的 fresh session — 只有用 --session 才是 continuity，否則每次都是乾淨的上下文
  • Big Pickle 模型輸出長度有限 — 複雜 HTML 可能被截斷，需要分段落餵任務
  • 沒有 stderr 區分錯誤 — exit code 0 不等於任務完成，還是要看輸出內容
  • 沒有直接檔案編輯回調 — OpenCode 只能透過 tool use 改檔案，Hermes 之間需要靠寫檔案中轉

9. 與 Hermes Agent 整合實戰

Hermes Agent 擔任 orchestrator，OpenCode 擔任 executor。
核心模式：prompt 定義目標 → --session 保持上下文 → fork 建立分支實驗，形成可持續的對話式專案執行鏈。

基本 Chain 模式

1
起始任務：Hermes 派發初始需求 
opencode run "在 opencodetest/ 建立一份 API 文件 index.html，包含 RESTful 路由表與 curl 範例" \
  --dir ~/Desktop/special_topic/opencodetest \
  --model opencode/big-pickle \
  --format json --title "建立 API 文件"
2
Hermes 從 JSON 輸出解析 sessionID，存入變數以串接下一個任務 
# 假設回傳 sessionID = ses_abc123...
opencode run "新增一章關於錯誤碼的表格" \
  --session ses_abc123...
3
需要實驗性修改時使用 --fork，不影響原始 session 
opencode run "改用紫色主題重新設計" \
  --fork --session ses_abc123...
4
用 session list --json 讓 Hermes 程式化判斷 session 狀態 
opencode session list --format json | jq '.[] | select(.title | contains("API"))'

實戰案例：Hermes 腳本自動化

#!/usr/bin/env bash
# Hermes 排程腳本：每日更新報告
SESSION_ID=$(opencode run --prompt "更新 index.html 的市場數據表格" \
  --dir ~/Desktop/special_topic/opencodetest \
  --model opencode/big-pickle --format json 2>&1 | grep sessionID | cut -d' ' -f2)

opencode run --prompt "將更新日誌寫入 CHANGELOG.md" \
  --session $SESSION_ID
關鍵技巧：每次 opencode run 都用 --format json 輸出，讓 Hermes 能程式化解析 sessionID、status、token 用量等資訊。Chain 的長度取決於模型 context window，建議每 5–8 次 iteration 後用 --fork 歸零 context。

10. 常見問題與解決方案（Limitations & Pitfalls）

問題原因解決方案
--prompt 沒有效果 未使用 opencode run，直接下 opencode --prompt 不會執行 完整指令：opencode run --prompt "..."
輸出被截斷 模型 context window 限制或 token 上限 拆分任務、使用 --fork 開新 session、或換更大 context 的模型（如 nemotron）
Session 遺失 手動刪除 ~/.config/opencode/sessions.db 或目錄重建 定期用 opencode export <id> 備份重要 session
模型無法存取 provider API 變更、斷線、或模型已下架 用 opencode models 確認可用清單，切換備用模型
檔案沒被修改 OpenCode 不知道目標目錄，或路徑錯誤 使用 --dir 明確指定目錄，prompt 中寫絕對/相對路徑
Session 無法繼續 session ID 拼錯、或資料庫損毀 用 opencode session list 確認正確 ID；備份 db 在 ~/.config/opencode/
Hermes 收不到 JSON --format json 輸出混在人類可讀文字中 用 2>&1 | grep '{' | tail -1 提取 JSON 行，或寫入暫存檔
Context 污染 長時間 session 累積太多歷史，模型開始混淆 定時用 --fork 開分支，只保留關鍵上下文；大任務拆成多個 session
